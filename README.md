# Лабораторная работа №3
## 1. Предметная область
Система предназначена для учета студентов в общежитии. В ней будут отслеживаться:
- Сведения о студентах (ФИО, факультет, курс, контактные данные)
- Информация о комнатах общежития (номер комнаты, корпус, количество мест, количество занятых мест)
- Данные о заселении и выселении студентов (дата заселения, дата выселения, статус проживания)
## 1. Цели системы
Система учета решает следующие задачи:
- Автоматизация учёта студентов, проживающих в общежитии
- Оптимизация процесса заселения
- Формирование отчётности по занятости комнат
- Повышение удобства для студентов
## 3. Пользователи системы и их роли.
В системе выделяются следующие категории пользователей:
**Комендант** общежития
- Отвечает за заселение и выселение студентов;
- Может просматривать и редактировать данные о студентах и комнатах;
- Формирует отчёты по занятости;
- Назначает комнаты студентам.
**Студент**
- Может подавать заявку на заселение 
- Просматривает информацию о своей комнате и сроках проживания.
## 4. Основные функции системы
Для _Коменданта_ общежития:
- Оформление заселения студента в комнату
- Оформление выселения студента
- Просмотр информации об общежитии
Для _студента_:
- Подача заявки на заселение
- Подача заявки на выселение(по собственному желанию)
- Подача заявки на перенаселение
- Просмотр комнат
## 5. Архитектура.
**Presentation Layer (слой представления)**
Предназначен для взаимодействия пользователей с системой. Например просмотр комнат студентом и заселение в эту комнату
**Business Logic Layer (слой бизнес-логики)**
Реализует правила работы: проверка наличия свободных мест, логика заселения/выселения.
**Data Layer (слой данных)**
Хранение информации в реляционной базе данных. База данных где хранятся таблицы: студентов, комнат в общежитии. 
Пример таблицы:
| № студента | эл.почта | № общежития | № комнаты |
| ---------- | -------- | ----------- | --------- |
| 204| Vasyapupkin13@mail.ru | 1 | 153 |
